# Plan: Unified Docker for All Worktrees

## Problem

Every git worktree created by Claude Code gets its own copy of `docker-compose.yml`. Running `make up` inside a worktree spins up containers that mount the worktree path, which is correct — but:

1. Multiple worktrees trying to use the same ports (8000, 5173, 5432, 6379) conflict
2. Container names clash (`finance-laravel`, `finance-postgres`)
3. Volume names clash (`pgdata`, `vendor`, `node_modules`)
4. Database state is not shared between worktrees (or dangerously is, if you're not careful)

## Goal

One `docker-compose.yml` at the project root, reusable from any worktree, with automatic isolation per worktree (ports, container names, volumes all namespaced).

---

## Solution: Root docker-compose with `.env` override

### Root `docker-compose.yml` changes

Replace hardcoded `./backend` and `./frontend` paths with env vars. Replace hardcoded container names and project name with `${COMPOSE_PROJECT_NAME}`.

```yaml
# docker-compose.yml (root — source of truth)
name: ${COMPOSE_PROJECT_NAME:-finance-app}

services:
  postgres:
    image: postgres:16-alpine
    container_name: ${COMPOSE_PROJECT_NAME:-finance-app}-postgres
    environment:
      POSTGRES_DB: ${POSTGRES_DB:-finance}
      POSTGRES_USER: ${POSTGRES_USER:-finance}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD:-finance}
      TZ: America/Sao_Paulo
    ports:
      - "${POSTGRES_PORT:-5432}:5432"
    volumes:
      - ${COMPOSE_PROJECT_NAME:-finance-app}-pgdata:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${POSTGRES_USER:-finance}"]
      interval: 5s
      timeout: 5s
      retries: 10
    networks: [finance-app]

  redis:
    image: redis:7-alpine
    container_name: ${COMPOSE_PROJECT_NAME:-finance-app}-redis
    ports:
      - "${REDIS_PORT:-6379}:6379"
    volumes:
      - ${COMPOSE_PROJECT_NAME:-finance-app}-redisdata:/data
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 5s
      timeout: 3s
      retries: 10
    networks: [finance-app]

  laravel:
    build:
      context: ${BACKEND_PATH:-./backend}
      dockerfile: Dockerfile
    container_name: ${COMPOSE_PROJECT_NAME:-finance-app}-laravel
    ports:
      - "${LARAVEL_PORT:-8000}:8000"
    volumes:
      - ${BACKEND_PATH:-./backend}:/var/www/html
      - ${COMPOSE_PROJECT_NAME:-finance-app}-vendor:/var/www/html/vendor
    depends_on:
      postgres: {condition: service_healthy}
      redis:    {condition: service_healthy}
    networks: [finance-app]

  vite:
    image: node:20-alpine
    container_name: ${COMPOSE_PROJECT_NAME:-finance-app}-vite
    working_dir: /app
    command: sh -c "npm install --no-audit --no-fund && npm run dev -- --host 0.0.0.0 --port 5173"
    environment:
      TZ: America/Sao_Paulo
      CHOKIDAR_USEPOLLING: "true"
    ports:
      - "${VITE_PORT:-5173}:5173"
    volumes:
      - ${FRONTEND_PATH:-./frontend}:/app
      - ${COMPOSE_PROJECT_NAME:-finance-app}-node_modules:/app/node_modules
    networks: [finance-app]

volumes:
  ${COMPOSE_PROJECT_NAME:-finance-app}-pgdata:
  ${COMPOSE_PROJECT_NAME:-finance-app}-redisdata:
  ${COMPOSE_PROJECT_NAME:-finance-app}-vendor:
  ${COMPOSE_PROJECT_NAME:-finance-app}-node_modules:

networks:
  finance-app:
    driver: bridge
```

> **Note**: Docker Compose doesn't interpolate variable names in `volumes:` keys. Use a `docker-compose.override.yml` approach instead (see below).

## Practical approach: override file per worktree

Since Docker Compose doesn't allow variable interpolation in volume names, the cleanest solution is:

### 1. Root `docker-compose.yml` stays as-is (path-based)

Paths use `${BACKEND_PATH}` and `${FRONTEND_PATH}` — these work fine.

### 2. Each worktree's `.env` file sets paths and project name

Create `.env.docker` in each worktree (not tracked by git — add to `.gitignore`):

```bash
# .env.docker (inside worktree, auto-generated by Makefile)
COMPOSE_PROJECT_NAME=finance-app-naughty-khayyam
BACKEND_PATH=/path/to/worktree/backend
FRONTEND_PATH=/path/to/worktree/frontend
POSTGRES_PORT=5432
REDIS_PORT=6379
LARAVEL_PORT=8000
VITE_PORT=5173
```

### 3. Root `Makefile` generates `.env.docker` automatically

```makefile
# Detect if inside a worktree or main repo
WORKTREE_NAME := $(shell basename $(CURDIR))
ROOT_DIR := $(shell git worktree list | head -1 | awk '{print $$1}')
COMPOSE_FILE := $(ROOT_DIR)/docker-compose.yml

.env.docker:
	@echo "COMPOSE_PROJECT_NAME=finance-$(WORKTREE_NAME)" > .env.docker
	@echo "BACKEND_PATH=$(CURDIR)/backend" >> .env.docker
	@echo "FRONTEND_PATH=$(CURDIR)/frontend" >> .env.docker
	@echo "POSTGRES_PORT=5432" >> .env.docker
	@echo "REDIS_PORT=6379" >> .env.docker
	@echo "LARAVEL_PORT=8000" >> .env.docker
	@echo "VITE_PORT=5173" >> .env.docker

up: .env.docker
	docker compose -f $(COMPOSE_FILE) --env-file .env.docker up -d

down:
	docker compose -f $(COMPOSE_FILE) --env-file .env.docker down

logs:
	docker compose -f $(COMPOSE_FILE) --env-file .env.docker logs -f laravel vite

artisan:
	docker compose -f $(COMPOSE_FILE) --env-file .env.docker exec laravel php artisan $(filter-out $@,$(MAKECMDGOALS))
```

### Worktree isolation note
Each worktree still gets its own database (separate `COMPOSE_PROJECT_NAME` → different Docker volume). This is intentional: a feature branch should not share DB state with main.

If you want to copy main's DB to a worktree: `make db-copy-from-main`.
